[ 배경 ]
EfficientNet-B3 backbone 선택의 근거를 실험적으로 입증하기 위해
ResNet-50 / DenseNet-121 / MobileNetV3-Large 와의 비교 실험 파이프라인을 추가.

실험 구조는 변수 분리를 위해 2단계로 설계:
  Phase 1 - Backbone 비교: 4개 backbone × CE (loss 변수 고정)
  Phase 2 - 메인 실험:     EfficientNet-B3 × CE vs CB-Focal

전체 epoch: 30 (ImageNet pretrained fine-tuning 기준 적정값)
모든 backbone: ImageNet-1k pretrained (timm) 으로 조건 통일

------------------------------------------------------------
  수정된 파일
------------------------------------------------------------

[1] model.py
  - SUPPORTED_BACKBONES dict 추가
      {
        "efficientnet_b3":       "efficientnet_b3",
        "resnet50":              "resnet50",
        "densenet121":           "densenet121",
        "mobilenetv3_large_100": "mobilenetv3_large_100",
      }
  - build_model(backbone, num_classes, pretrained) 팩토리 함수 추가
      timm.create_model 을 backbone 키로 선택해 생성
      지원하지 않는 backbone 입력 시 ValueError 발생
  - build_efficientnet_b3 는 하위 호환성 유지용 래퍼로 유지

[2] main.py
  - import 변경: build_efficientnet_b3 → build_model, SUPPORTED_BACKBONES
  - --backbone CLI 인자 추가
      default: "efficientnet_b3"
      choices: SUPPORTED_BACKBONES 의 키 목록
  - --save_name default 변경: "best_model.pth" → None (미지정 시 자동 생성)
  - INFO 출력에 backbone 항목 추가
  - 모델 생성: build_efficientnet_b3() → build_model(backbone=args.backbone, ...)
  - 저장 파일명 자동 생성
      best model : outputs/best_model_{backbone}_{loss_type}.pth
      history    : outputs/history_{backbone}_{loss_type}.json

------------------------------------------------------------
  신규 생성된 파일
------------------------------------------------------------

[3] run_experiments.sh
  - 5번의 실험을 순차 실행하는 쉘 스크립트
  - Phase 1: backbone 비교 (CE 고정)
      efficientnet_b3      + ce
      resnet50             + ce
      densenet121          + ce
      mobilenetv3_large_100 + ce
  - Phase 2: 메인 실험
      efficientnet_b3      + cb_focal (beta=0.999, gamma=2.0)
  - 모든 실험 완료 후 analyze_results.py 자동 호출
  - DATA_DIR 환경변수로 데이터 경로 변경 가능 (기본값: ./data)
  - 사용법: bash run_experiments.sh
            DATA_DIR=/path/to/data bash run_experiments.sh

[4] analyze_results.py
  - outputs/ 의 history_*.json 을 모두 읽어 결과를 분석
  - 파일명 파싱: history_{backbone}_{loss_type}.json 형식
  - 생성 결과물
      outputs/summary.csv
          컬럼: experiment, backbone, loss_type,
                best_macro_f1, final_train_loss, final_val_loss, epochs
          best_macro_f1 내림차순 정렬
      outputs/plots/f1_comparison.png
          실험별 Best Macro F1 막대 그래프 (CE: 파란색, CB-Focal: 주황색)
      outputs/plots/learning_curves.png
          Val Macro F1 / Val Loss 학습 곡선 (CB-Focal 은 점선 구분)
  - 사용법: python analyze_results.py --output_dir ./outputs
